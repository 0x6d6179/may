# contributing

## reporting issues

found a bug? please open an issue. feature ideas welcome too. we review issues regularly and appreciate detailed context.

## submitting changes

1. fork the repository
2. create a branch for your change (`git checkout -b fix/my-change`)
3. make your changes
4. push your branch and open a pull request

## development setup

see `docs/dev.md` for local setup and build instructions.

## code style

keep it consistent with the existing codebase:

- lowercase user-facing strings
- avoid fmt.Println — use structured logging where available
- keep commands flat and simple
- minimize external dependencies

thanks for contributing.
